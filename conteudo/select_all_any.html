Há operadores lógicos que são utilizados e podemos citar os operadores ANY e ALL. Ambos comparam um valor escalar com resultado de uma subconsulta que retorne uma única coluna e com a utilização de qualquer um dos operadores de comparação. Veja abaixo

= igual a
> maior do que
>= maior ou igual a
< menor do que
<= menor ou igual a
< > diferente de

Operador ANY

  • retorna um valor booleano como resultado
  • retorna verdadeiro se qualquer um dos valores da subconsulta atender à condição

ANY significa que a condição será verdadeira se operação for verdadeira para qualquer um dos valores do intervalo

Operador ALL

  • retorna verdadeiro se todos os valores da subconsulta atenderem à condição
  • é usado com SELECT, WHERE e HAVING

ALL significa que a condição será verdadeira se operação for verdadeira para todos os valores no intervalo.

Comando SQL - DQL - operador ANY

Qual pizza Especial tem o mesmo preço ou é mais barata que alguma pizza Tradicional?

SELECT FROM INNER JOIN WHERE
* pizza tipo ON pizza.id_tipo=tipo.id_tipo tipo ='Especial' AND preco <= ANY (SELECT preco FROM pizza INNER JOIN tipo ON pizza.id_tipo=tipo.id_tipo WHERE tipo ='Tradicional')

Camarão 49

Comando SQL - DQL - operador ALL

Qual pizza Especial é mais cara que todas as pizzas do tipo Tradicional?

SELECT FROM INNER JOIN WHERE
* pizza tipo ON pizza.id_tipo=tipo.id_tipo tipo ='Especial' AND preco > ALL (SELECT preco FROM pizza INNER JOIN tipo ON pizza.id_tipo=tipo.id_tipo WHERE tipo ='Tradicional')

Resultado:

Camarão 49
